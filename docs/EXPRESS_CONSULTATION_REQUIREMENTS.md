# Требования к функционалу экспресс-консультаций

## Цель и назначение

**Экспресс-консультация** — это инструмент продаж, который позволяет консультанту:
- Провести быстрый разбор клиента (15-30 минут)
- Встроить скрипт продаж в формат бесплатного разбора
- Продать полную консультацию или парную консультацию
- Помочь неопытным консультантам проводить эффективные консультации через готовые фразы и персонализированный подход

**Ключевая идея:** Разбор — это не продажа, разбор — это продукт. Но через правильную структуру разбора человек сам приходит к желанию купить полную консультацию.

## Основная логика работы

1. **Создание запроса на экспресс-консультацию** в приложении
2. **n8n выполняет базовый расчет** САЛ-кодов (как сейчас)
3. **n8n отправляет результат в Directus** (создает профиль с `book_information`)
4. **Приложение получает эту информацию** и настраивает этапы, вопросы и персонализированный контент
5. **Консультант проводит консультацию** по структурированным шагам с готовыми фразами

## Структура консультации (4 шага)

### 1. Точка А (Текущая ситуация)
**Задача:** Столкнуть человека с реальностью, чтобы он осознал, что его текущая ситуация его не устраивает.

- Вопросы о том, что не получается, что не устраивает
- Выявление проблем, сомнений, препятствий
- Фиксация всех проблем для использования при продаже

### 2. Точка Б (Хотелки и видение)
**Задача:** Вдохновить человека и показать, что он может прийти к жизни своей мечты.

- Вопросы о желаниях, целях, мечтах
- Углубление в истинную мотивацию
- Визуализация идеальной ситуации

### 3. Ресурсы
**Задача:** Показать, что ресурсы есть, но либо не понятно как их реализовать, либо чего-то не хватает.

- Анализ доступных ресурсов клиента
- Показ того, что может дать полная консультация
- Формирование понимания, что недостающие ресурсы — это наш продукт

### 4. Закрытие и продажа
**Задача:** Продать полную консультацию или парную консультацию.

- Резюме консультации (точка А → точка Б)
- Оценка важности для клиента (1-10)
- Выбор формата: полная консультация или парная
- Завершение и фиксация результата продажи

## Ключевые требования к персонализации

### 1. Персонализация на основе САЛ-кодов

**Источник данных:**
- Трактовки кодов берутся из поля `book_information` в коллекции `Profiles`
- Формат: JSON с описанием каждого кода (personality, connector, realization, generator, mission)
- Opener берется из поля `raw_json.opener` профиля

**Синтез всех кодов:**
- Система должна учитывать **синтез всех кодов**, а не только отдельные
- Если у человека доминирующие коды (например, 4 5 4 4 4), нужно учитывать доминацию
- Персонализация должна строиться на **полном видении профиля человека**

### 2. Стиль общения

**Определение стиля на основе коннектора:**
- **Direct (1, 3, 5, 7, 9):** Прямое общение, "ты", короткие фразы
- **Soft (2, 4, 6, 8):** Мягкое общение, "вы", развернутые фразы
- **Balanced (11, 22):** Сбалансированный стиль

**Важно:** Клиент с кодом коннектора 2 и клиент с кодом коннектора 8 по-разному воспринимают информацию. Формулировать информацию для них нужно по-разному.

### 3. Универсальность запросов

**Критически важно:** Запросы могут быть самыми разными:
- Не только про деньги
- Про семью, отношения, детей
- Про бизнес, карьеру
- Про здоровье, самореализацию
- Про творчество, признание
- И многое другое

Система должна **адаптироваться под любой запрос**, а не быть заточена только под финансовые цели.

### 4. Готовые фразы для консультантов

**Проблема:** Многие консультанты теряются, когда начинают проводить консультации из-за неопытности.

**Решение:** Система должна предоставлять:
- **Готовые фразы для установления контакта** (динамичные, актуальные, живые)
- **Персонализированные вопросы** для каждого шага
- **Готовые фразы для каждого этапа** консультации
- **Персонализированный оффер** на основе проблем и целей клиента

**Важно:** Фразы должны быть **персональными под клиента**, а не общими шаблонами. Они должны учитывать синтез всех кодов и стиль общения.

### 5. Анализ ресурсов с точки зрения САЛ

На этапе "Ресурсы" система должна показать:
- Может ли клиент прийти к точке Б с точки зрения САЛ
- За счет каких ресурсов (на основе кодов)
- Что может мешать (искажения, вызовы)
- Коротко и тезисно, чтобы вызвать желание узнать больше

## Технические требования

### 1. Интеграция с n8n
- При создании экспресс-консультации автоматически вызывается `/api/calc` для базового расчета
- n8n создает профиль с `book_information` в Directus
- Приложение получает эти данные и использует для персонализации

### 2. Хранение данных
- Консультация сохраняется в коллекции `consultations` с типом `"express"`
- Шаги сохраняются в коллекции `consultation_steps`
- Каждый шаг содержит: `step_type`, `question`, `response`, `selected_options`

### 3. Восстановление состояния
- При перезагрузке страницы все шаги восстанавливаются
- Сохраненные ответы отображаются в формах
- Можно вернуться к любому шагу и изменить ответ

### 4. UX требования
- **Аккордеон-схема:** Один шаг открыт, остальные свернуты
- **Максимум кнопок, минимум текста:** Для медленно печатающих консультантов
- **Визуальная индикация:** Завершенные шаги отмечены зеленой галочкой
- **Мобильная адаптивность:** Работает на всех устройствах

## Ожидаемый результат

**Цель:** Система должна помогать продавать там, где консультант сам не сможет, просто благодаря правильному подходу.

**Критерии успеха:**
- Консультант получает персонализированные фразы под конкретного клиента
- Вопросы и опции адаптируются под стиль общения клиента
- Система учитывает синтез всех кодов, а не только отдельные
- Консультация работает для любых запросов (не только деньги)
- Неопытные консультанты могут эффективно проводить консультации
- Высокая конверсия в продажу (цель: 99%)

## Примеры персонализации

### Пример 1: Доминирующие коды
Если у клиента коды 4 5 4 4 4:
- Учитывается доминация кода 4
- Персонализация строится на характеристиках кода 4
- Но также учитываются остальные коды для полной картины

### Пример 2: Разные стили общения
- **Коннектор 2:** "Как вам хочется? К чему вы хотите прийти?"
- **Коннектор 8:** "Как хочется? К чему ты хочешь прийти?"

### Пример 3: Разные запросы
- **Запрос про семью:** Вопросы и опции про отношения, детей, гармонию
- **Запрос про бизнес:** Вопросы и опции про доход, клиентов, масштабирование
- **Запрос про здоровье:** Вопросы и опции про самочувствие, энергию, баланс

## Дополнительные требования

1. **Opener из базового расчета:** Использовать opener из профиля для начала консультации
2. **Фразы для установления контакта:** Динамичные, актуальные (погода, сезон, события)
3. **Анализ ресурсов:** Показывать, может ли клиент прийти к цели с точки зрения САЛ
4. **Персонализированный оффер:** Использовать конкретные проблемы и цели клиента из консультации
5. **Готовые фразы в каждом шаге:** Отдельная секция с фразами для консультанта
