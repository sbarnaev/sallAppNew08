# Настройка долгих токенов в Directus

Для того чтобы токены были долгими или вечными, нужно настроить Directus через переменные окружения:

## Переменные окружения Directus

Добавьте в `.env` файл Directus или в настройки контейнера:

```env
# Время жизни access token (по умолчанию 15 минут)
# Установите очень большое значение (например, 10 лет в секундах)
AUTH_TOKEN_TTL=315360000

# Время жизни refresh token (по умолчанию 7 дней)
# Установите очень большое значение (например, 10 лет в секундах)
AUTH_REFRESH_TOKEN_TTL=315360000

# Или для вечных токенов (не рекомендуется для production)
# AUTH_TOKEN_TTL=0
# AUTH_REFRESH_TOKEN_TTL=0
```

## Значения в секундах

- 1 год = 31,536,000 секунд
- 10 лет = 315,360,000 секунд
- 0 = бесконечный срок (не рекомендуется)

## Примечание

Время жизни cookies в приложении уже установлено на 10 лет. Но само время жизни токена в Directus настраивается через переменные окружения Directus.
